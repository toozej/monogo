# Plan: Rearchitect CLI from Flags to Sub-commands

## Current Architecture

All modes are controlled by flags on a single root command (`root.go`). The current invocation pattern is:

```bash
go-find-archived-gh-actions [flags]
```

Key flags that act as "mode selectors":
- `--workflow <path>` ‚Äî scan a specific workflow file
- `--workflows-dir <dir>` ‚Äî scan a directory of workflow files
- `--repos-dir <dir>` ‚Äî scan multiple repos under a directory
- *(default, no path flag)* ‚Äî scan the current repo's `.github/workflows/`

Check-type flags (currently all run together in a single `processWorkflows` call):
- `--check-outdated` ‚Äî check for outdated action versions
- `--stale` / `--stale-days` ‚Äî check for stale/deprecated actions
- Archived action checking is **always on** (no flag to disable it)

Output/action flags:
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue
- `--write` / `-W` ‚Äî write updates to workflow files (requires `--check-outdated`)

Global flags:
- `--token` / `-t` ‚Äî GitHub token
- `--debug` / `-d` ‚Äî debug logging
- `--verbose` / `-v` ‚Äî verbose output

Existing sub-commands: `version`, `man`

## Proposed Sub-command Architecture

```
go-find-archived-gh-actions [global flags] <command> [command flags]
```

### Commands

| Command | Description | Replaces |
|---------|-------------|----------|
| `archived` (default) | Scan current repo for archived actions | root command with no path flags |
| `workflow` | Scan a specific workflow file | `--workflow` flag |
| `dir` | Scan a directory of workflow files | `--workflows-dir` flag |
| `repos` | Scan multiple repos in a directory | `--repos-dir` flag |
| `outdated` | Check for outdated action versions (implies archived check too) | `--check-outdated` flag |
| `stale` | Check for stale/deprecated actions (implies archived check too) | `--stale` / `--stale-days` flags |
| `version` | Print version info | *(already exists)* |
| `man` | Generate man pages | *(already exists)* |

### Invocation Examples

**Before:**
```bash
go-find-archived-gh-actions --token $GH_TOKEN --verbose
go-find-archived-gh-actions --workflow ci.yml --check-outdated
go-find-archived-gh-actions --workflows-dir ./workflows --stale --stale-days 180
go-find-archived-gh-actions --repos-dir ~/src --notify --create-issue
go-find-archived-gh-actions --check-outdated --write
```

**After:**
```bash
go-find-archived-gh-actions archived --token $GH_TOKEN --verbose
go-find-archived-gh-actions workflow ci.yml --outdated
go-find-archived-gh-actions dir ./workflows --stale --stale-days 180
go-find-archived-gh-actions repos ~/src --notify --create-issue
go-find-archived-gh-actions outdated --write
```

### Flag Placement by Command

#### Global (persistent) flags ‚Äî available on all commands
- `--token` / `-t` ‚Äî GitHub token
- `--debug` / `-d` ‚Äî debug logging
- `--verbose` / `-v` ‚Äî verbose output

#### `archived` ‚Äî scan current repo
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue
- *(archived check always runs ‚Äî this is the core behavior)*

#### `workflow <path>` ‚Äî scan a single workflow file
- Positional arg: path to workflow file
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue

#### `dir <path>` ‚Äî scan a directory of workflow files
- Positional arg: path to workflows directory
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue

#### `repos <path>` ‚Äî scan multiple repos
- Positional arg: path to base directory containing repos
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue

#### `outdated` ‚Äî check for outdated actions (runs archived check + version check)
- (operates on current repo by default)
- `--workflow <path>` ‚Äî scope to specific workflow file
- `--workflows-dir <dir>` ‚Äî scope to directory
- `--repos-dir <dir>` ‚Äî scope to multiple repos
- `--write` / `-W` ‚Äî write updates to workflow files (pin to latest SHA)
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue

#### `stale` ‚Äî check for stale/deprecated actions (runs archived check + stale check)
- (operates on current repo by default)
- `--workflow <path>` ‚Äî scope to specific workflow file
- `--workflows-dir <dir>` ‚Äî scope to directory
- `--repos-dir <dir>` ‚Äî scope to multiple repos
- `--stale-days` ‚Äî number of days before considered stale (default 365)
- `--notify` ‚Äî send notifications
- `--create-issue` ‚Äî create a GitHub issue

### Design Decisions & Tradeoffs

1. **`archived` as default command**: Running `go-find-archived-gh-actions` with no subcommand runs `archived` (current default behavior). This preserves backward compatibility for the most common use case. We accomplish this by making `rootCmd.Run` delegate to the `archived` subcommand when no subcommand is specified (using `rootCmd.RunE` with `cmd.Help()` fallback, or using the `SetArgs` trick on `rootCmd`).

2. **`outdated` and `stale` as top-level commands with `--workflow`/`--dir`/`--repos-dir` flags**: Rather than nesting (e.g., `outdated workflow ci.yml`), we keep path scoping as flags on these commands. This avoids a deep nesting hierarchy (`outdated/repos/dir/workflow`) and keeps the CLI flat and ergonomic. The `--write` flag only makes sense on `outdated`, so it lives there exclusively.

3. **Archived checking is always on**: The core value of this tool is detecting archived actions. Both `outdated` and `stale` commands run the archived check first, then add their respective additional checks. This matches current behavior where archived checking is unconditional.

4. **Backward compatibility**: The old flag-based interface (`--workflow`, `--workflows-dir`, `--repos-dir`, `--check-outdated`, `--stale`, `--write`, `--notify`, `--create-issue`) will be **removed** ‚Äî this is a major version break. No deprecation shim will be added.

## Implementation Steps

1. **Create shared state struct** ‚Äî Extract shared command state (token resolution, verbose/debug flags, config) into a struct or helper to avoid duplicating initialization across 6 subcommand `Run` functions.

2. **Create `archived.go`** ‚Äî Move the default-run logic (scan current repo for archived actions) into an `archivedCmd`. This is the simplest subcommand and will be the default when no subcommand is given.

3. **Create `workflow_cmd.go`** ‚Äî Move `--workflow` logic into a `workflowCmd` that takes a positional path arg.

4. **Create `dir.go`** ‚Äî Move `--workflows-dir` logic into a `dirCmd` that takes a positional path arg.

5. **Create `repos_cmd.go`** ‚Äî Move `--repos-dir` / `runReposMode` logic into a `reposCmd` that takes a positional path arg.

6. **Create `outdated.go`** ‚Äî Move `--check-outdated` + `--write` logic into an `outdatedCmd`. Accepts `--workflow`, `--workflows-dir`, `--repos-dir` as scoping flags plus `--write`, `--notify`, `--create-issue`.

7. **Create `stale_cmd.go`** ‚Äî Move `--stale` / `--stale-days` logic into a `staleCmd`. Accepts `--workflow`, `--workflows-dir`, `--repos-dir` as scoping flags plus `--stale-days`, `--notify`, `--create-issue`.

8. **Refactor `root.go`** ‚Äî Remove the old mode-selector flags, `rootCmdRun`, and `runReposMode`. Wire up the new subcommands. Add default-run delegation to `archived` when no subcommand is given.

9. **Update tests** ‚Äî Add command-level tests for each subcommand. Existing helper function tests remain. Verify `archived` runs as the default subcommand.

10. **Update docs and examples** ‚Äî `.pre-commit-hooks.yaml`, `find-archived-repos-workflow-example.yaml`, `README.md`.

## Resolved Design Decisions

1. **Backward compatibility**: Major version break ‚Äî old flags will be removed entirely, no deprecation shim.
2. **Path scoping on `outdated`/`stale`**: `--workflow`/`--dir`/`--repos-dir` will be flags on `outdated` and `stale` commands (flat, ergonomic).
3. **Default command name**: `archived` (specific to what it does, clearer than generic `check`).
